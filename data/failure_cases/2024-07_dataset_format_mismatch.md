# 실패 사례: 오픈소스 h5 데이터셋과 우리 zarr 마스터 암 포맷 불일치

- **시도한 작업**: 오픈소스 모방학습 코드(HDF5 기반 데이터 로더)를 우리
  마스터 암 데이터(zarr 포맷)에 그대로 연결
- **결과**: 데이터 로더의 키 구조(observation/state, action 등)와 우리
  zarr 스키마의 필드명/축 순서가 달라, 단순 포맷 변환만으로는 동작하지
  않음. 변환 코드를 새로 작성하는 데만 1주일 이상 소요되었고, 이후에도
  타임스탬프 정렬 문제로 데이터 일부가 손상됨
- **원인**: "데이터 포맷이 다르다"는 사실만 확인하고, 실제 필드 단위
  매핑표(블록 다이어그램) 작성 없이 바로 코드 수정에 들어감
- **결론**: 오픈소스 코드를 도입할 때는 실물/시뮬레이터 적용 전에
  반드시 (1) 데이터 스키마 필드 단위 매핑표, (2) 타임스탬프/주파수
  정합성 확인을 먼저 마친 뒤 이식 가능 여부를 판단할 것. 매핑표 없이
  바로 코드 수정하는 시도는 반복하지 말 것
