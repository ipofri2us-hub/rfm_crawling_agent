# 실패 사례: Diffusion Policy 적용 시 제어 주파수(Hz) 급락

- **시도한 기법**: Diffusion Policy 기반 액션 생성 (디노이징 스텝 다수 적용)
- **환경**: 실제 마스터 암 + 로컬 GPU 추론 서버
- **결과**: 디노이징 스텝을 논문 기본값(예: 50~100 step)으로 적용 시 1회 액션
  생성에 200ms 이상 소요되어 제어 주파수가 5Hz 이하로 급락. 목표 제어
  주파수(예: 20Hz, 50ms) 대비 4배 이상 느려져 실시간 제어 불가능
- **원인**: 디노이징 스텝 수와 추론 latency가 선형적으로 비례하는데, 논문에서
  제시한 성공률은 step 수를 줄이면 함께 하락함 (속도-정확도 trade-off)
- **결론**: Diffusion 계열 정책을 도입할 때는 반드시 "최대 latency 50ms"
  제약 안에서 동작하는 step 수(보통 4~10 step, DDIM 등 가속 샘플러 사용)로
  먼저 사전 검토할 것. 기본 설정값 그대로 실물 로봇에 올리는 시도는
  반복하지 말 것
